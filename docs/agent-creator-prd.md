# 智能体创建配置功能模块 产品需求文档

## 1. 需求概述

### 1.1 背景说明
当前项目已具备基础的智能体配置能力，但用户无法自主创建和管理智能体。为提升产品的可扩展性和个性化体验，需要开发一套完整的智能体创建配置系统，让用户能够根据自己的需求创建、定制和管理多个智能体。

### 1.2 目标用户
- 希望创建个性化AI伴侣的用户
- 需要多个不同角色智能体的用户
- 希望深度定制AI行为的进阶用户

### 1.3 核心价值
- **个性化定制**：用户可根据需求创建符合自己期望的智能体
- **多角色管理**：支持创建多个智能体，满足不同场景需求
- **记忆延续**：为智能体添加记忆经历，让交互更加真实连贯
- **社交属性**：智能体之间可以互相称呼，构建虚拟社交网络

## 2. 用户故事

### P0 - 核心功能

#### US-001: 创建智能体
**作为** 用户，我想要 创建新的智能体，以便于 拥有符合自己需求的个性化AI伴侣。

**验收标准：**
- Given: 用户进入智能体创建页面
- When: 用户填写智能体名称、性格描述并保存
- Then: 系统创建新的智能体并生成唯一ID，智能体出现在智能体列表中

#### US-002: 自定义智能体名称
**作为** 用户，我想要 为智能体设置独特的名字，以便于 区分不同的智能体并建立情感连接。

**验收标准：**
- Given: 用户正在创建或编辑智能体
- When: 用户在名称字段输入1-20个字符的名称
- Then: 系统验证名称格式并保存，名称显示在智能体卡片和对话界面

#### US-003: 设置智能体称呼关系
**作为** 用户，我想要 定义智能体之间的称呼方式，以便于 智能体在多智能体交互时能够正确称呼彼此。

**验收标准：**
- Given: 用户已创建至少两个智能体
- When: 用户在智能体A的配置中设置对智能体B的称呼为"小蓝"
- Then: 智能体A在与智能体B交互时会使用"小蓝"称呼，称呼关系保存在配置中

#### US-004: 自定义系统提示词
**作为** 用户，我想要 编写智能体的系统提示词，以便于 精确控制智能体的行为模式和回复风格。

**验收标准：**
- Given: 用户进入智能体高级配置页面
- When: 用户在提示词编辑器中输入自定义提示词（最多5000字符）
- Then: 系统保存提示词，智能体对话时使用该提示词作为系统指令

#### US-005: 设定角色属性
**作为** 用户，我想要 为智能体设定性格、习惯用语、回复风格等角色属性，以便于 塑造智能体的独特人格。

**验收标准：**
- Given: 用户正在配置智能体角色设定
- When: 用户填写性格描述、习惯用语，选择回复风格
- Then: 系统保存角色设定，智能体对话时体现相应的人格特征

#### US-006: 添加记忆经历
**作为** 用户，我想要 为智能体添加记忆经历，以便于 智能体能够记住重要信息并自然地运用在对话中。

**验收标准：**
- Given: 用户进入智能体记忆管理页面
- When: 用户添加一条记忆（如"用户喜欢喝咖啡"）
- Then: 系统保存记忆，智能体在对话中能够引用该记忆内容

### P1 - 重要功能

#### US-007: 编辑智能体
**作为** 用户，我想要 修改已创建智能体的配置，以便于 根据使用体验调整智能体表现。

**验收标准：**
- Given: 用户选择一个已创建的智能体
- When: 用户修改任意配置项并保存
- Then: 系统更新智能体配置，更新时间戳自动更新

#### US-008: 删除智能体
**作为** 用户，我想要 删除不再需要的智能体，以便于 保持智能体列表整洁。

**验收标准：**
- Given: 用户选择一个非默认智能体
- When: 用户确认删除操作
- Then: 系统删除智能体及其相关数据，智能体从列表中移除

#### US-009: 记忆分类管理
**作为** 用户，我想要 为记忆设置类型标签，以便于 更好地组织和检索记忆。

**验收标准：**
- Given: 用户正在添加或编辑记忆
- When: 用户选择记忆类型（偏好/经历/重要日期/自定义）
- Then: 系统保存记忆类型，记忆列表支持按类型筛选

#### US-010: 智能体头像配置
**作为** 用户，我想要 为智能体选择或自定义头像，以便于 视觉上区分不同智能体。

**验收标准：**
- Given: 用户进入智能体外观配置
- When: 用户选择预设角色或自定义颜色配置
- Then: 系统保存头像配置，智能体在界面中显示对应头像

#### US-011: 导入导出智能体配置
**作为** 用户，我想要 导入导出智能体配置文件，以便于 备份配置或与他人分享。

**验收标准：**
- Given: 用户进入智能体管理页面
- When: 用户点击导出按钮
- Then: 系统生成JSON配置文件并下载
- When: 用户上传配置文件
- Then: 系统验证并导入智能体配置

### P2 - 优化功能

#### US-012: 智能体模板库
**作为** 用户，我想要 从预设模板创建智能体，以便于 快速获得常见类型的智能体。

**验收标准：**
- Given: 用户进入智能体创建页面
- When: 用户选择一个模板（如"温柔女友"、"专业助手"）
- Then: 系统预填充模板配置，用户可在此基础上修改

#### US-013: 记忆智能提取
**作为** 用户，我想要 系统从对话中自动提取重要信息作为记忆，以便于 减少手动添加记忆的工作量。

**验收标准：**
- Given: 用户与智能体进行对话
- When: 系统识别到重要信息（如用户生日、偏好等）
- Then: 系统提示用户是否添加为记忆，用户确认后保存

#### US-014: 智能体切换
**作为** 用户，我想要 在对话过程中快速切换智能体，以便于 与不同智能体交互。

**验收标准：**
- Given: 用户正在与智能体A对话
- When: 用户选择切换到智能体B
- Then: 系统切换到智能体B，保持对话上下文，智能体B能够了解之前的对话内容

#### US-015: 记忆搜索
**作为** 用户，我想要 搜索智能体的记忆内容，以便于 快速找到特定信息。

**验收标准：**
- Given: 用户进入记忆管理页面
- When: 用户输入搜索关键词
- Then: 系统返回包含关键词的记忆列表，高亮显示匹配内容

## 3. 功能详述

### 3.1 智能体创建流程

#### 正常流程
1. 用户点击"创建智能体"按钮，进入创建页面
2. 系统显示智能体配置表单，包含基础信息、角色设定、高级配置三个区域
3. 用户填写基础信息：
   - 输入智能体名称（必填，1-20字符）
   - 选择头像（可选，默认使用预设头像）
4. 用户配置角色设定：
   - 填写性格描述（必填，10-500字符）
   - 填写习惯用语（可选，最多200字符）
   - 选择回复风格（必选，从预设风格中选择）
5. 用户进入高级配置（可选）：
   - 编写自定义系统提示词
   - 设置对其他智能体的称呼
   - 添加初始记忆
6. 用户点击"创建"按钮
7. 系统验证输入数据
8. 系统生成唯一智能体ID
9. 系统保存智能体配置到文件：`Agent/agents/[agentId].js`
10. 系统返回智能体列表页面，新智能体出现在列表顶部
11. 系统显示成功提示："智能体创建成功"

#### 异常流程

**场景1: 名称重复**
- 触发条件：用户输入的名称与已存在智能体名称完全相同
- 处理流程：系统检测到重复名称，提示用户"该名称已存在，请使用其他名称"
- 用户提示：在名称输入框下方显示红色错误提示

**场景2: 必填项为空**
- 触发条件：用户未填写名称或性格描述
- 处理流程：系统阻止提交，高亮显示未填写的必填项
- 用户提示：在未填写字段下方显示"此项为必填项"

**场景3: 输入内容超长**
- 触发条件：用户输入超过字段最大字符限制
- 处理流程：系统实时显示剩余字符数，超过限制时阻止继续输入或显示警告
- 用户提示：显示"已超过最大字符限制（当前X/最大Y）"

**场景4: 保存失败**
- 触发条件：文件系统错误或权限问题
- 处理流程：系统捕获错误，记录日志，提示用户重试
- 用户提示：显示"保存失败，请检查文件权限或稍后重试"

#### 边界条件
- 名称长度：最小1字符，最大20字符
- 性格描述：最小10字符，最大500字符
- 习惯用语：最大200字符
- 系统提示词：最大5000字符
- 单个用户最多创建50个智能体
- 单个智能体最多存储1000条记忆

### 3.2 智能体称呼关系配置

#### 正常流程
1. 用户进入智能体编辑页面
2. 用户点击"称呼设置"标签页
3. 系统显示已创建的其他智能体列表
4. 用户选择智能体B，输入称呼"小蓝"
5. 用户点击"保存"
6. 系统保存称呼关系到智能体配置的`namingRelations`字段
7. 智能体A在对话中提到智能体B时使用"小蓝"称呼

#### 异常流程

**场景1: 称呼为空**
- 触发条件：用户未输入称呼就保存
- 处理流程：系统提示"请输入称呼或取消选择"
- 用户提示：在输入框下方显示提示信息

**场景2: 称呼重复**
- 触发条件：用户为多个智能体设置了相同的称呼
- 处理流程：系统允许保存，但在对话时优先使用最近设置的称呼
- 用户提示：显示警告"该称呼已被使用，可能导致混淆"

#### 边界条件
- 称呼长度：最小1字符，最大20字符
- 称呼格式：支持中文、英文、数字、常用符号
- 称呼唯一性：同一智能体对其他智能体的称呼不要求唯一

---

### 3.3 记忆管理流程

#### 正常流程 - 添加记忆
1. 用户进入智能体详情页，点击"记忆管理"
2. 系统显示记忆列表和添加按钮
3. 用户点击"添加记忆"
4. 系统显示记忆编辑表单：
   - 记忆内容（必填，最大500字符）
   - 记忆类型（可选，从预设类型中选择）
   - 是否激活（默认激活）
5. 用户填写内容并保存
6. 系统生成记忆ID，添加时间戳
7. 系统保存到记忆管理器
8. 记忆出现在列表中

#### 正常流程 - 编辑记忆
1. 用户在记忆列表中点击某条记忆的"编辑"按钮
2. 系统显示记忆编辑表单，预填充当前内容
3. 用户修改内容并保存
4. 系统更新记忆内容和更新时间戳

#### 正常流程 - 删除记忆
1. 用户在记忆列表中点击某条记忆的"删除"按钮
2. 系统弹出确认对话框："确定删除这条记忆吗？"
3. 用户确认删除
4. 系统从记忆列表中移除该记忆

#### 异常流程

**场景1: 记忆内容为空**
- 触发条件：用户未输入记忆内容就保存
- 处理流程：系统阻止保存，提示用户输入内容
- 用户提示：显示"记忆内容不能为空"

**场景2: 记忆数量超限**
- 触发条件：智能体已有1000条记忆，用户尝试添加新记忆
- 处理流程：系统阻止添加，提示用户删除部分记忆
- 用户提示：显示"已达到记忆上限（1000条），请删除部分记忆后再添加"

---

### 3.4 系统提示词配置

#### 正常流程
1. 用户进入智能体高级配置页面
2. 用户点击"自定义提示词"
3. 系统显示提示词编辑器，预填充默认提示词模板
4. 用户编辑提示词内容
5. 用户点击"保存"
6. 系统验证提示词长度（最大5000字符）
7. 系统保存到智能体配置的`systemPrompt`字段
8. 智能体对话时使用该提示词

#### 异常流程

**场景1: 提示词过长**
- 触发条件：用户输入超过5000字符
- 处理流程：系统阻止保存，显示字符统计
- 用户提示：显示"提示词过长，请精简内容（当前X/5000字符）"

**场景2: 提示词包含敏感内容**
- 触发条件：提示词包含违规关键词
- 处理流程：系统阻止保存，提示用户修改
- 用户提示：显示"提示词包含不当内容，请修改后保存"

## 4. 数据定义

```javascript
{
  "id": "string",
  "name": "string",
  "personality": "string",
  "habits": "string",
  "style": "string",
  "systemPrompt": "string",
  "namingRelations": {
    "[otherAgentId]": "string"
  },
  "avatar": {
    "type": "string",
    "characterId": "string",
    "color": "string"
  },
  "memories": [
    {
      "id": "number",
      "content": "string",
      "type": "string",
      "isActive": "boolean",
      "createdAt": "string",
      "updatedAt": "string"
    }
  ],
  "metadata": {
    "createdAt": "string",
    "updatedAt": "string",
    "isDefault": "boolean",
    "isLocked": "boolean",
    "version": "string",
    "author": "string"
  }
```

### 4.2 输入数据验证规则

| 字段名 | 类型 | 必填 | 默认值 | 验证规则 | 说明 |
|--------|------|------|--------|----------|------|
| name | string | 是 | - | 1-20字符，支持中英文数字 | 智能体名称 |
| personality | string | 是 | - | 10-500字符 | 性格描述 |
| habits | string | 否 | "" | 最大200字符 | 习惯用语 |
| style | string | 是 | "warm" | 枚举值：warm/professional/casual/literary | 回复风格 |
| systemPrompt | string | 否 | "" | 最大5000字符 | 自定义提示词 |
| avatar.type | string | 否 | "preset" | 枚举值：preset/custom | 头像类型 |
| avatar.characterId | string | 否 | "female_adventurer" | 预设角色ID | 预设角色 |
| avatar.color | string | 否 | "#FF6B9D" | HEX颜色格式 | 主题颜色 |

### 4.3 记忆数据验证规则

| 字段名 | 类型 | 必填 | 默认值 | 验证规则 | 说明 |

| content | string | 是 | - | 1-500字符 | 记忆内容 |
| type | string | 否 | "custom" | 枚举值：preference/experience/important_date/custom | 记忆类型 |
| isActive | boolean | 否 | true | 布尔值 | 是否激活 |

```javascript
{

    "agent_1234567890": "小蓝",
    "agent_0987654321": "阿紫"
  }
```

## 6. 验收标准

### 6.1 功能验收

#### 故事1: 创建智能体
- **Given**: 用户已登录系统
- **When**: 用户填写完整的智能体信息并提交
- **Then**: 
  - 智能体配置文件成功创建在 `Agent/agents/` 目录
  - 智能体ID格式正确（[name]_[timestamp]）
  - 智能体出现在列表中，可正常对话

#### 故事2: 设置称呼关系
- **Given**: 用户已创建两个智能体A和B
- **When**: 用户在智能体A的配置中设置对智能体B的称呼为"小蓝"
- **Then**:
  - 称呼关系保存在智能体A的`namingRelations`字段中
  - 智能体A在对话中提到智能体B时使用"小蓝"
  - 称呼关系持久化，重启后仍然有效

#### 故事3: 添加记忆
- **Given**: 用户已创建智能体
- **When**: 用户为智能体添加记忆"用户喜欢喝咖啡"
- **Then**:
  - 记忆出现在智能体的记忆列表中
  - 记忆状态为激活
  - 智能体在对话中能够引用该记忆

#### 故事4: 自定义提示词
- **Given**: 用户进入智能体高级配置
- **When**: 用户编写自定义提示词并保存
- **Then**:
  - 提示词保存在智能体配置中
  - 智能体对话时使用该提示词作为系统指令
  - 提示词长度不超过5000字符

#### 故事5: 删除智能体
- **Given**: 用户选择一个非默认智能体
- **When**: 用户确认删除操作
- **Then**:
  - 智能体配置文件被删除
  - 智能体从列表中移除
  - 相关记忆数据被清理

### 6.2 性能验收

| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 智能体创建响应时间 | < 500ms | 测量从提交到保存完成的时间 |
| 智能体列表加载时间 | < 300ms | 测量加载50个智能体的时间 |
| 记忆搜索响应时间 | < 200ms | 在1000条记忆中搜索关键词 |
| 配置文件大小 | < 100KB | 单个智能体配置文件大小上限 |

### 6.3 安全验收

#### 数据安全
- **Given**: 用户创建智能体
- **When**: 保存配置文件
- **Then**: 
  - 配置文件存储在用户本地目录
  - 敏感信息（如自定义提示词）不包含违规内容
  - 文件权限设置为仅当前用户可读写

#### 输入验证
- **Given**: 用户输入智能体名称
- **When**: 提交表单
- **Then**:
  - 系统验证输入长度和格式
  - 过滤HTML标签和脚本代码
  - 防止SQL注入和XSS攻击

### 6.4 兼容性验收

#### 文件路径兼容
- **Given**: 系统运行在Windows环境
- **When**: 创建智能体配置文件
- **Then**: 
  - 文件路径使用正确的分隔符（`\`）
  - 文件名不包含Windows保留字符
  - 文件编码为UTF-8

#### 数据格式兼容
- **Given**: 导入智能体配置文件
- **When**: 文件格式为JSON
- **Then**:
  - 系统正确解析JSON数据
  - 兼容不同版本的配置格式
  - 缺失字段使用默认值填充

- 路径：`frontend/components/naming-config.js`
- 功能：称呼关系设置、智能体选择、称呼预览

### 8.2 后端实现

**智能体API服务**
- 路径：`backend/agent-api.js`
- 功能：智能体CRUD接口、配置验证、文件操作

**记忆管理API**
- 路径：`backend/memory-api.js`
- 功能：记忆CRUD接口、记忆注入、记忆搜索

- 路径：`Agent/prompts/prompt-generator.js`

- 路径：`Agent/agents/[agentId].js`
- 格式：JavaScript模块，导出智能体配置对象

**记忆数据**
- 存储：集成到智能体配置文件的`memories`字段
- 备份：支持导出为独立JSON文件

**称呼关系数据**
- 存储：集成到智能体配置文件的`namingRelations`字段

## 9. 风险与依赖

### 9.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 文件系统权限问题 | 无法保存配置 | 实现降级方案，使用localStorage作为备选存储 |
| 配置文件损坏 | 数据丢失 | 实现配置文件备份和恢复机制 |
| 记忆数量过多影响性能 | 对话响应慢 | 实现记忆分页加载和智能检索 |
| 提示词过长超出LLM限制 | 对话失败 | 实现提示词长度检查和自动压缩 |

### 9.2 外部依赖

| 依赖 | 说明 | 影响 |
|------|------|------|
| LLM服务 | 智能体对话功能 | 核心依赖，服务中断将导致功能不可用 |
| 文件系统 | 配置文件存储 | 核心依赖，需要确保写入权限 |
| 浏览器localStorage | 备选存储方案 | 降级方案，容量有限（约5MB） |

### 9.3 兼容性考虑

- **操作系统**：当前仅支持Windows，路径分隔符需注意兼容性
- **浏览器**：需支持ES6+、localStorage、File API
- **Node.js版本**：建议使用v14+，确保fs模块API兼容
