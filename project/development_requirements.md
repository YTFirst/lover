## 1. 项目概述

### 1.1 产品定义

心伴（HeartMate）是一款 Windows 桌面端 AI 虚拟伴侣应用。用户在首次使用时自行设定角色的名字、性格、外观和 System Prompt，然后通过文字对话与角色建立情感连接。应用采用沉浸式设计——对话页面以角色立绘为背景，营造"和角色面对面聊天"的感觉。

### 1.2 MVP核心体验

```
首次启动 → 用户设定角色（名字/性格/外观/System Prompt）
    → 进入对话主页（角色立绘背景 + 聊天窗口）
    → 对话过程中自动生成每日记忆 / 每周提炼记忆
    → 角色通过记忆"记住"用户的一切
```

### 1.3 设计原则

- **用户定义一切**：角色设定、System Prompt、记忆规则全部交给用户，我们只提供默认模板
- **沉浸式对话**：对话页面以角色立绘为背景，不是冰冷的聊天工具
- **记忆即陪伴**：参考 OpenClaw 的记忆模式，每日记忆 + 每周提炼，让角色真正"记住"用户

### 1.4 MVP范围

**包含：**
- ✅ 首次角色设定（名字、性格、外观、System Prompt）
- ✅ AI文字对话（流式输出、打字机效果）
- ✅ 角色立绘背景的沉浸式对话界面
- ✅ 长期记忆系统（每日记忆MD + 每周提炼MD）
- ✅ 用户自定义 System Prompt（提供默认模板）
- ✅ 设置系统（API配置、对话设置、记忆管理、隐私）

**不包含（后期规划）：**
- ❌ 语音通话 / TTS语音合成
- ❌ Live2D实时动画（MVP使用静态立绘+表情切换）
- ❌ 好感度 / 心情 / 状态系统
- ❌ 桌面宠物模式
- ❌ 角色选择页（只支持1个角色）
- ❌ UGC社区 / 角色分享
- ❌ 付费系统

### 5.2 AI对话引擎

#### 5.2.1 功能描述

核心对话功能。用户与角色进行文字对话，角色的人格完全由用户的 System Prompt 定义。对话界面以角色立绘为背景，营造沉浸式体验。

#### 5.2.2 对话流程

```
用户输入文字
    → 前端校验（非空、长度限制）
    → 渲染用户气泡（乐观更新）
    → 通过IPC发送到Main Process
    → Main Process组装消息:
        1. 读取用户的 System Prompt
        2. 替换变量: {{charName}} → 角色名, {{currentTime}} → 当前时间
        3. 读取相关记忆文件内容 → 替换 {{relevantMemories}}
        4. 加载最近50条历史消息
    → 组装最终 messages 数组 [system, ...history, user]
    → 调用LLM API（stream: true）
    → 流式返回 → 逐字渲染AI气泡
    → 完成后异步: 更新当日记忆文件
```

#### 5.2.3 Prompt组装逻辑

```
最终发送给LLM的messages数组:

[
  {
    role: "system",
    content: 用户的System Prompt（变量已替换）
  },
  ...最近50条历史消息,
  {
    role: "user",
    content: 用户当前输入
  }
]
```

**变量替换规则：**

| 变量 | 替换为 | 示例 |
|------|--------|------|
| `{{charName}}` | 角色名字 | "小雪" |
| `{{currentTime}}` | 当前日期时间 | "2026年4月15日 周三 下午3:26" |
| `{{relevantMemories}}` | 相关记忆内容 | 见记忆系统章节 |

> **核心原则**: 我们不修改用户的 System Prompt，只做变量替换。角色的行为完全由用户定义。

### 5.3 记忆系统（OpenClaw模式）

#### 5.3.1 设计理念

参考 OpenClaw 的记忆方案，核心思路是：

- **每日记忆**: 每天生成一个 Markdown 文件，记录当天对话中的关键信息
- **每周提炼**: 每周生成一个提炼过的 Markdown 文件，总结本周的重要记忆
- **注入Prompt**: 对话时将相关记忆内容注入 System Prompt 的 `{{relevantMemories}}` 变量

这种方式简单、透明、用户可编辑，比黑盒式的向量数据库更可控。

#### 5.3.2 记忆文件结构

```
%AppData%/HeartMate/
└── memories/
    ├── daily/
    │   ├── 2026-04-13.md
    │   ├── 2026-04-14.md
    │   ├── 2026-04-15.md
    │   └── ...
    └── weekly/
        ├── 2026-W15.md    ← 4月第3周
        ├── 2026-W16.md    ← 4月第4周
        └── ...
```

#### 5.3.3 每日记忆文件格式

文件名: `YYYY-MM-DD.md`

```markdown
# 2026-04-15 对话记忆

## 关键信息
- 用户名字叫小明
- 用户在北京工作，是一名程序员
- 用户今天加班到很晚，看起来很累

## 重要事件
- 用户今天完成了一个大项目的上线

## 情感状态
- 用户今天整体情绪偏疲惫，但完成项目后有些开心

## 对话摘要
今天主要聊了工作上的事情，用户分享了他加班的经历...
```

#### 5.3.4 每周提炼记忆文件格式

文件名: `YYYY-WXX.md`

```markdown
# 2026年第15周 记忆总结 (4.7 - 4.13)

## 用户画像
- 名字: 小明
- 职业: 程序员，在北京工作
- 兴趣: 喜欢看动漫、打游戏
- 宠物: 有一只猫叫豆豆
- 性格: 偶尔会吐槽工作，但整体乐观

## 本周重要事件
- 周一: 用户升职了，很开心
- 周三: 用户和同事发生了矛盾，心情不好
- 周五: 用户周末打算在家休息

## 持续关注
- 用户最近工作压力比较大
- 用户提到下周有一个重要的面试

## 关系进展
- 用户开始更主动地分享日常了
- 聊天频率稳定，每天都会来对话
```

#### 5.3.5 记忆生成流程

**每日记忆生成：**

```
触发时机: 当天第一次对话结束后 / 当天最后一次对话结束后 / App关闭时

流程:
1. 收集当天所有对话内容
2. 调用LLM（使用专用记忆提取Prompt），生成当日记忆MD内容
3. 如果当天已有记忆文件，追加/更新（而非覆盖）
4. 保存到 memories/daily/YYYY-MM-DD.md
```

**记忆提取Prompt（内置，用户不可修改）：**

```markdown
请根据以下对话内容，生成当天的记忆总结，以Markdown格式输出。

要求：
1. 提取关于用户的关键信息（名字、工作、喜好、生活等）
2. 记录重要事件
3. 感知用户的情感状态
4. 写一段简短的对话摘要
5. 如果与之前的记忆有矛盾或更新，标注出来
6. 语言简洁，每条信息一行

以下是之前的记忆摘要（供参考）：
{{previousMemories}}

今天的对话内容：
{{todayConversations}}

请输出Markdown格式的记忆总结。
```

**每周提炼记忆生成：**

```
触发时机: 每周一第一次打开App时 / 手动触发（设置中"立即提炼本周记忆"）

流程:
1. 读取本周所有每日记忆文件
2. 读取上周的提炼记忆（作为上下文）
3. 调用LLM，生成本周提炼记忆
4. 保存到 memories/weekly/YYYY-WXX.md
```

**每周提炼Prompt（内置）：**

```markdown
请根据以下每日记忆，生成本周的记忆总结。

要求：
1. 更新用户画像（整合新信息，修正过时信息）
2. 列出本周重要事件
3. 标注需要持续关注的事项
4. 总结关系进展
5. 去重：如果某信息已经在之前的周记忆中存在且无变化，不需要重复

上周记忆总结：
{{lastWeekSummary}}

本周每日记忆：
{{thisWeekDailyMemories}}

请输出Markdown格式的本周记忆总结。
```

#### 5.3.6 记忆注入对话


## 6. 数据模型

### 6.1 SQLite表结构

```sql
-- 用户设置（KV存储）
CREATE TABLE settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL
);

-- 角色设定（只有一个角色）
CREATE TABLE character (
  id INTEGER PRIMARY KEY CHECK (id = 1),  -- 单例，永远只有一条
  name TEXT NOT NULL,
  avatar_path TEXT,             -- 头像图片路径
  portrait_path TEXT,           -- 立绘图片路径
  system_prompt TEXT NOT NULL,  -- 用户定义的System Prompt
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 对话会话（按天自动创建）
CREATE TABLE conversations (
  id TEXT PRIMARY KEY,
  date TEXT NOT NULL,           -- YYYY-MM-DD，每天一个会话
  started_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  message_count INTEGER DEFAULT 0
);

-- 消息记录
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  conversation_id TEXT NOT NULL,
  role TEXT NOT NULL,           -- 'user' | 'assistant'
  content TEXT NOT NULL,
  timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
  metadata TEXT,                -- JSON: {model, tokens, duration}
  FOREIGN KEY (conversation_id) REFERENCES conversations(id)
);

-- 对话历史全文搜索
CREATE VIRTUAL TABLE messages_fts USING fts5(
  content,
  content=messages,
  content_rowid=rowid
);

-- 记忆文件索引（辅助管理，实际记忆内容在MD文件中）
CREATE TABLE memory_files (
  id TEXT PRIMARY KEY,
  type TEXT NOT NULL,           -- 'daily' | 'weekly'
  date_key TEXT NOT NULL,       -- '2026-04-15' 或 '2026-W15'
  file_path TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 7.2 LLM API调用规范

```typescript
interface LLMConfig {
  baseUrl: string;        // e.g. "https://api.openai.com/v1"
  apiKey: string;
  model: string;          // e.g. "gpt-4o-mini"
  maxTokens: number;      // 默认 1024
  temperature: number;    // 默认 0.8
}
```

**调用流程：**
1. Renderer发送消息 → IPC → Main Process
2. Main Process读取: character.system_prompt → 替换变量
3. 读取记忆文件 → 替换 `{{relevantMemories}}`
4. 加载最近N条历史消息
5. 组装 messages 数组 → 调用LLM API（stream: true）
6. 流式返回 → IPC推送 → Renderer逐字渲染
7. 完成后异步: 保存消息到DB → 检查是否需要更新当日记忆

## 附录B: 关键交互状态

### B.1 首次启动流程

```
App启动
  → 检查 character 表是否有数据
    → 无: 路由到 /setup
      → Step1: 设置名字、选择头像和立绘
      → Step2: 编辑 System Prompt（预填默认模板）
      → 点击"开始聊天"
      → 保存到 character 表
      → 路由到 /（对话主页）
    → 有: 直接路由到 /
```

### B.2 对话中的表情切换

```
AI回复内容
  → 简单情感分析（关键词匹配）
    → 包含"哈哈"/"嘻嘻"/"♪" → happy
    → 包含"呜"/"难过"/"..." → sad
    → 包含"哼"/"才不是" → angry
    → 包含"害羞"/"脸红" → shy
    → 包含"诶"/"啊"/"！" → surprised
    → 默认 → default
  → 立绘区域切换对应表情图（如果有）
  → 200ms交叉淡入淡出过渡
```

> MVP阶段立绘表情切换为可选功能，取决于是否有对应表情图片。如果没有，立绘保持不变。

### B.3 错误处理

| 场景 | 处理方式 |
|------|---------|
| API Key未配置 | 设置页提示"请先配置API Key"，对话区显示引导卡片 |
| API调用失败 | 气泡显示错误提示，提供"重试"按钮 |
| 网络断开 | 顶部黄色提示条"网络连接已断开"，输入框禁用 |
| 数据库损坏 | 自动备份旧库 + 重建空库 + 通知用户 |
| 记忆生成失败 | 静默失败，不影响对话功能，日志记录错误 |
